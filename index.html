What is React.js?

  • Javascript Library
  • 'View' in Model-View-Controller
  • Built by Facebook & Instagram
  • Open Source
  • Reactive data flow
  • Uses JSX (Javascript XML) wait what?

What is JSX?

  • 'Javascript XML'
  • Not required for React.js
  • People use React BECAUSE of JSX
  • JSX is compiled into Javascript for the browser to understand

Why React.js?

  • Simple
  • Declarative
  • Composable 'Components'
  • One-way data flow

We built React to solve one problem: building large applications with data that changes over time.

{how}

3 Step Process

  • Create variable
  • Return HTML (or other response)
  • Render results

Lets see how to do it

<!--DOCTYPE HTML-->
<html>
  <head>
    <meta charset="utf-8">
    <title>React Hello World</title>
  </head>
  <body>
    <!--
    You include React,
    then the JSXTransformer which turns JSX into JavaScript 
    and then your React code.
    
    In production, you'll want to not use the JSXTransformer
    and should instead look at using something like Gulp, Grunt or
    WebPack to compile JSX into JavaScript. Also, check out:
    http://facebook.github.io/react/docs/tooling-integration.html
    -->
    <script src="http://fb.me/react-0.12.2.min.js"></script>
    <script src="http://fb.me/JSXTransformer-0.12.2.js"></script>
    
    <!--
    This is where you link to your React code. Can be .js or .jsx 
    extension, doesn't really matter.
    -->
    <script src="scripts.jsx" type="text/jsx"></script>
  </body>
</html>

Flexibility of components

var CommentBox = React.createClass({
  render: function() {
    return (
      <div className="commentBox">
        <h1>Comments</h1>
        <CommentList  data={this.props.data} />
        <CommentForm />
      </div>
    );
  }
});

var CommentList = React.createClass({
  render: function() {
    var commentNodes = this.props.data.map(function (comment) {
      return (
        <Comment author={comment.author}>
          {comment.text}
        </Comment>
      );
    });
    return (
      <div className="commentList">
        {commentNodes}
      </div>
    );
  }
});

var CommentForm = React.createClass({
  render: function() {
    return (
      <div className="commentForm">
        Hello, world! I am a CommentForm.
      </div>
    );
  }
});

var Comment = React.createClass({
  render: function() {
    return (
      <div className="comment">
        <h2 className="commentAuthor">
          {this.props.author}
        </h2>
        {this.props.children}
      </div>
    );
  }
});

var data = [
  {author: "Pete Hunt", text: "This is one comment"},
  {author: "Jordan Walke", text: "This is another comment"}
];

React.render(
  <CommentBox data={data}/>,
  document.getElementById('example')
);

React.js vs. Angular.js

  • Virtual DOM or DOM re-render?
  • Angular: Application, Controller, Directives
  • React: Class, Render
  • Angular: $scope
  • React: this.props
  • One way data flow learning curve

'Angular is very opinionated, which can be great if it fits your development scenario. React gives you freedom and simplicity, but lacks the declarative power of Angular.'
